Abstract
--------

MCU_SLEEP defined. Fluke 45/17B 3 mv @ 10 OHM = 300 uA
PRR=98%,  MCU_DUTY_CYCLE=4%,   RDC=0.2%,  Current consumption=220uA

MCU_SLEEP undefined. Fluke 45/17B 51 mv @ 10 OHM --> 5.1 mA
PRR=100%,  MCU_DUTY_CYCLE=N/A,  RDC=0.8%,  Current consumption=5.1mA


Experimental description
------------------------
ContikiMAC MCU_SLEEP/RDC experiment. Report node sends pkts every 30 sec
to Sink RPL root and UDP server. Received pkts are loggen on console
and saved for analyses.

This work addresses the challenge to duty cycle MCU and not only to duty
cycle the radio (RDC). Here we use ContikiMAC(RDC in Contimi-OS. The challenge
is to keep RDC with a minimum MCU use. The total power consumption can be
calculated.

P(tot) = RDC*RADIO_POWER + MCU_DC*MCU_POWER (1) 

The challenges is to keep RDC functional with very little MCU power
enabling effective sleep modes. Turns off the timers normally needed by
the RDC. In this work a method of synchronizing fast clock with slow
realtime (RTC) clock is demonstrated.

Experimental results
---------------------
UDP packet payload example SEQ=1266 MCU_SLEEP=36479 RADIO_ON=88 SEC=37995

SEQ        - UDP packet number sent (sender)
SEC        - test time elapsed in seconds (sender)
MCU_SLEEP  - time in seconds MCU been in deep sleep (sender)
RADIO_ON   - time in seconds radio has been ON  (sender)
pkts       - UDP packets received (receiver)

Note. We lost 30 pkt before RPL converged. 

Estimating PDR when RPL has converged. 
1266-30 = 1236
Sink has received 1214 pkts.

PRR = 1214/1236 -->                   PRR =  98%
MCU USE = 1 - 36479/37995  MCU_DUTY_CYCLE =   4%
RADIO USE = 88/37995                RDC   = 0.2%                

Calculation
------------
From datasheets:
Radio RX Power = 10mA
MCU Busy       =  5mA

Using the formula in 1.
P(tot) = RDC*RADIO_POWER + MCU_DC*MCU_POWER (1)
p(tot) = 10*0.002 + 5*0.04  --> 0.22 uA

Discussion
----------
The current consumption was verified with FLUKE 45/FLUKE 17B
multimeters.

Interesting to note the MCU power use is a factor 10 bigger. 
The total cost of RDC should be evaluated including timers
and MCU usage. In this work the RDC is modified to use a
cheap RTC.
